# SecOps

Security controls, compliance tracking, secret management, audits.

What belongs here:

- Control notes: one per implemented security control, with what it protects,
  how it is verified, and its audit evidence location.
- Secret rotation schedule and procedures.
- Compliance mapping: which framework requirements are covered by which
  controls (keep framework-specific details in one note per framework).
- Audit and pentest findings with remediation status (mirror open items to
  `_Index/Open Loops.md`).

Suggested frontmatter: `type: control | rotation | compliance | finding`.
