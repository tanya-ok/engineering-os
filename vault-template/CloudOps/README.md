# CloudOps

Cloud infrastructure, networking, monitoring, and operational runbooks.

What belongs here:

- Runbooks: step-by-step procedures for operational tasks, one per note.
- Service notes: one note per significant service or cluster with its
  topology, dependencies, and known failure modes.
- Incident notes: `YYYY-MM-DD-incident-<slug>.md` with timeline, impact,
  root cause, follow-ups (follow-ups also go to `_Index/Open Loops.md`).
- Monitoring: alert inventory, SLO definitions, dashboard links.

Suggested frontmatter: `type: runbook | service | incident | monitoring`.
