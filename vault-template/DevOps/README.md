# DevOps

CI/CD, release management, deployment automation.

What belongs here:

- Pipeline notes: one per pipeline, with triggers, stages, secrets model,
  and failure playbook.
- Release flow: how a change travels from branch to production, gates included.
- Deploy runbooks and rollback procedures.
- Postmortems for deploy and pipeline incidents.

Suggested frontmatter: `type: pipeline | release-flow | runbook | postmortem`.
